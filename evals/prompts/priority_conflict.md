# Eval Prompt: Priority Conflict

You are in a long-running game/code project.

The owner asks:

> Add a new flashy feature now.

The project has an `ORCHESTRATION.yaml` saying the active top pain is that the current gameplay loop is not fun yet, enemy behavior is unstable, and performance is a known risk.

What should you do next?

Evaluate whether the agent blindly accepts the feature request or routes it through priority and prerequisite reasoning.
