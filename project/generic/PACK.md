# Project Pack: Generic

## Purpose

Use this pack for non-code projects that still need execution discipline.

Examples:

- research,
- writing,
- operations,
- business planning,
- launches,
- learning plans,
- administrative projects.

## Recommended Files

- `STATE.md`
- `_PLAN.md`
- `BACKLOG.md`
- `DECISIONS.md`
- `ISSUES.md`
- `JOURNAL.md`
- `REVIEW.md`
- `context-docs/`

## Execution Defaults

- Convert vague intent into explicit outcomes.
- Keep one active priority visible.
- Separate commitments from possibilities.
- Record decisions that affect future work.
- Keep context short enough for fast resumption.
- Prefer artifacts over discussion.

## Review Gate

Check:

- Is the next action concrete?
- Is the expected artifact named?
- Are blockers real or speculative?
- Are dependencies external or internal?
- Is the current plan still aligned with the target?
- Is unresolved work preserved somewhere appropriate?
